C1 - Recenser et identifier les ressources numériques

GLPI est un outil qui permet de recenser et identifier des ressources d'un parque informatique

Pour recenser, on va installer les agents GLPI sur les machines cibles puis les lier a notre GLPI hôte. Dans un premier temps, on télécharge l'agent sois directement sur github sois avec wget https://github.com/glpi-project/glpi-agent/releases si on est sur linux

Après l'installation, on va l'installer avec sudo perl glpi-agent-1.17-linux-installer.pl et renseigner l'adresse ip de notre GLPI hôte quand l'url d'un serveur GLPI est demandé. Si tout s'est bien réalisé la machine apparaitra dans la liste des machines sur le GLPI hôte

Réponses aux incidents, demandes d’assistance et d’évolution

C7 - Collecter, suivre et orienter des demandes

Nous avons utilisé les tickets GLPI pour recenser les problèmes rencontrés durant le hackathon.

La gestion des demandes utilisateurs consiste à traiter les incidents et les requêtes à l’aide d’un outil de gestion de ticket comme ici GLPI. Elle permet d’assurer un suivi efficace des problèmes signalés, depuis leur apparition jusqu’à leur résolution.

Dans un premier temps, cela implique de recueillir les problèmes rencontrés par les utilisateurs comme par exemple un problème d’infrastructure Ces informations sont ensuite enregistrées sous formes de ticket dans GLPI, en précisant des éléments importants tels que les identités de l’utilisateur, la description du problème, la priorité et la catégorie.

Ensuite, le suivi des demandes doit être assuré. Cette étape consiste à mettre à jour régulièrement les tickets en fonction de leur avancement Des commentaires peuvent être ajoutés afin de garder une trace des actions réalisées. Le suivi permet de vérifier que chaque demande est bien prise en charge et traitée dans des délais adaptés.

Ainsi, la gestion des demandes via GLPI contribue à une meilleure organisation du support, tout en garantissant une communication claire avec les utilisateurs et une résolution plus rapide des incidents.

Après un certain laps de temps, les données sur google analytics seront mise à jour, pour nous permettre d'observer les passages sur notre site.

C17 - Déployer un service

Dans le cadre de notre groupe, nous avons eu l’opportunité d’héberger notre site en ligne au travers de 2 différents services, Microsoft Azure et Github Pages.

Apache

Apache est un programme qui permet la création d’un serveur web, en d’autres mots il permet d’afficher une page si on met l’adresse ip de notre machine en navigateur. Pour cela, dans le cas de linux on fait la commande “sudo apt install apache2” pour installer notre serveur web apache. Les fichiers de notre serveur se situent ensuite dans /var/www/html, c’est le DocumentRoot de notre serveur web.

Azure

Grâce à l’offre Microsoft Azure for student nous avons 100 dollars de crédits offerts à la création de notre compte Microsoft Azure mais également d’avoir accès à des petites VM gratuites, nous avons simplement installer apache sur cette VM ainsi que d’y téléverser nos documents html, css, js et cette vm sera en continue ouverte, il suffira simplement d’aller sur l’adresse IP de la VM puis nous verrons notre site, c’est d’ailleurs grâce à ce service que nous avons hébergé notre GLPI en ligne.

Github pages

Nous devons d’abord faire un dépôt git, et y ajouter les documents html, css, js, nous allons par suite passer par le service github pages et choisir la branche qu’on souhaite mettre en ligne puis github s’occupe de nous fournir un lien pour notre site mis en ligne.
